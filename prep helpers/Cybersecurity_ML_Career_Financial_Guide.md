#### Bug Bounty Methodology (Step-by-Step)
```
1. RECON (find attack surface):
   ├── Subdomain enumeration (subfinder, amass, assetfinder)
   ├── Port scanning (nmap, masscan)
   ├── Technology fingerprinting (Wappalyzer, whatweb)
   ├── Directory brute-forcing (ffuf, dirsearch)
   └── Google dorking (site:target.com filetype:pdf etc.)

2. VULNERABILITY HUNTING (by bug type):
   ├── IDOR: Change IDs in API requests, access other users' data
   ├── XSS: Inject payloads in all input fields, URL params, headers
   ├── SSRF: Find URL fetch features, try internal network access
   ├── Auth Bypass: Test session management, token manipulation
   ├── SQL Injection: Test all parameters with sqlmap + manual
   ├── Business Logic: Think like a user — what SHOULDN'T be possible?
   └── API Abuse: Test rate limits, mass assignment, broken access control

3. REPORTING (critical for getting paid):
   ├── Clear title describing impact
   ├── Step-by-step reproduction
   ├── Screenshots/video proof
   ├── Impact assessment (what can attacker do?)
   ├── Suggested remediation
   └── Professional tone (you're helping them, not attacking)
```
